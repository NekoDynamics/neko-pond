# 品牌系统

## 中文

### 品牌定位

NekoDynamics 是面向 iPadOS 的 AI 宠物交互系统组织。品牌表达应当安静、精密、技术化，强调实时交互、环境计算和动物感知，而不是休闲娱乐或卡通陪伴。

### 视觉方向

| 维度 | 标准 |
| --- | --- |
| 色彩 | 深色基底、低饱和蓝绿、柔和高光 |
| 对比 | 清晰但不刺眼，避免高糖度色彩 |
| 图形 | 几何、抽象、流体、环境化 |
| 材质 | 玻璃感、水面、微粒、柔和景深 |
| 留白 | 保持呼吸感，避免密集控件 |

### 字体建议

- 首选 Apple 系统字体：SF Pro、SF Pro Rounded、SF Mono。
- 标题使用简洁中性字重，避免装饰性字体。
- 数值、调试和性能数据使用等宽字体。
- 中文排版优先稳定行高和留白，不追求视觉噱头。

### 图标方向

- 使用线性、几何、低细节图标。
- 图标应表达系统能力：行为、渲染、触摸、环境。
- 避免猫脸、玩具、夸张鱼形或儿童化轮廓。

### 深色环境语言

- 背景应接近夜间水域、深层玻璃或暗色空间。
- 高光用于引导注意，而不是制造装饰。
- 渐变应服务于空间深度和运动轨迹。
- 视觉噪声必须可控，避免干扰猫的追踪行为。

### 动效语言

- 动效应短、轻、连续，强调响应的可信度。
- 运动应包含自然缓入、微扰和非重复节奏。
- 避免弹跳、夸张缩放和奖励式反馈。
- 任何动效都必须尊重帧稳定性和触摸延迟。

### UI 哲学

- 系统状态优先，装饰次之。
- 设置界面安静清晰，运行界面尽量不可见。
- 控件只在需要时出现，避免干扰互动生态。
- 所有品牌表达都应服务于沉浸、观察和长期稳定使用。

## English

### Brand Positioning

NekoDynamics is an AI pet interaction systems organization for iPadOS. The brand should feel calm, precise, and technical, emphasizing real-time interaction, ambient computing, and animal perception rather than casual entertainment or cartoon companionship.

### Visual Direction

| Dimension | Standard |
| --- | --- |
| Color | Dark base, low-saturation blue-green, soft highlights |
| Contrast | Clear but not harsh, no candy-like palettes |
| Graphics | Geometric, abstract, fluid, environmental |
| Material | Glass, water surface, particles, soft depth of field |
| Space | Generous spacing, no dense control surfaces |

### Typography

- Prefer Apple system fonts: SF Pro, SF Pro Rounded, SF Mono.
- Use neutral weights for headings; avoid decorative typefaces.
- Use monospace type for values, diagnostics, and performance data.
- Chinese layouts should prioritize stable line height and spacing over visual tricks.

### Iconography

- Use linear, geometric, low-detail icons.
- Icons should represent system capabilities: behavior, rendering, touch, environment.
- Avoid cat faces, toys, exaggerated fish shapes, or childlike silhouettes.

### Dark Ambient Language

- Backgrounds should evoke night water, deep glass, or dark spatial fields.
- Highlights guide attention instead of decorating the interface.
- Gradients should support depth and motion trails.
- Visual noise must be controlled to avoid disturbing feline tracking behavior.

### Motion Language

- Motion should be short, light, and continuous, emphasizing believable response.
- Movement should include natural easing, micro-variation, and non-repeating rhythm.
- Avoid bouncing, exaggerated scaling, and reward-like feedback.
- Every motion decision must respect frame stability and touch latency.

### UI Philosophy

- System state first, ornament second.
- Settings should be quiet and legible; runtime UI should be nearly invisible.
- Controls appear only when needed and should not disturb the interaction ecosystem.
- Brand expression should serve immersion, observation, and long-session stability.
